Criação do repositório- É quando criamos um espaço no GitHub para guardar nosso projeto e tipo uma pasta online 

 Clone do repositório- É copiar o repositório do GitHub para o seu computador para poder mexer nos arquivos localmente

 Criação de branch- Serve para testar mudanças sem mexer direto na versão principal.

Registro de commits- Commit é salvar as alterações feitas no projeto

Envio das alterações (push)- É quando enviamos para o GitHub as mudanças que fizemos no nosso computador.

- Atualização do repositório local (pull)- É puxar as atualizações do GitHub para o seu computador, deixando tudo atualizado.

Criação de Issues- As Issues servem para organizar tarefas, relatar problemas ou sugerir melhorias no projeto.

 Organização com Milestone- Milestone é usada para agrupar várias Issues dentro de um objetivo maior, como uma meta ou versão do projeto.

 Gerenciamento com Project- O Project ajuda a organizar as tarefas em formato de quadro, facilitando ver o que está pendente, em andamento ou concluído.

 Abertura de Pull Request- É quando você pede para juntar as alterações de uma branch com a branch principal. Serve para revisar antes de aceitar as mudanças.

 Merge- É quando as alterações da branch são realmente unidas à branch principal, finalizando o processo.
